\n\nYou can create a Web page on your desktop computer but nobody can see it but\nyou. If your want other people to be able to see your Web pages, you need\nan account on a computer that has a Web server. Nowadays most people have\ntheir own computers on their desks, but normally they don't have Web servers\nand anyway you don't want the whole world coming into your desktop computer\nto see your web page because (a) it's not designed for that, and (b) who\nknows what else they might see. And (c) for security reasons, Web servers\nshould be managed by professionals. Most institutions have big central\nshared computers for this purpose, which usually have\na Unix-like operating\nsystem such as Linux. You need an account on one of these so you can put\nyour web pages there. If you don't have access to such a computer, you can get\na low-cost account on a service like Panix.com.\n\n

\n\nYou can still create Web pages on your own computer and look at them with\nyour computer's Web browser, but for other people to see them, you have to\nupload them to the \"big\" computer that has the Web browser. The rest of\nthis document is about how to create your first Web page.\n\n

1. Creating a Web Page

\n\nThis page was typed by hand. Anybody can do this, you don't need any\nspecial \"web creation\" tools or HTML editors, and the pages you make can be\nviewed from any browser. To see how this page was made, choose\nView Source (or View Page Source, or View Document Source) in your\nbrowser's menu (or — in at least Chrome and Firefox — Ctrl-U on\nyour keyboard). A simple web page like this one is just plain text with\nHTML commands (markup) mixed in. HTML commands (properly called \"tags\")\nthemselves are plain text.\n\n

\n\nWhen you're just learning and want to experiment, you can do everything on\nyour PC. Create a new directory (\"folder\") for your website, and then put\nthe web-page files (HTML plus any pictures) in it. Use NotePad or other\nplain-text editor (not word processor) on your PC to create your \"home page\",\na file named\nindex.html, which you can view locally with your Web\nbrowser. (You can also use a word processors such as Word or WordPad if you\nsave in \"plain text\", \"text\", \"text document\", or \"text document MS-DOS\nformat\".)\nLater I'll explain how you can install your web site on\nthe Internet.\n\n

\n\nOnce you've made your \"home page\" (index.html) you can add more pages to\nyour site, and your home page can link to them.\n\n

2. HTML Syntax

\n\nWeb pages are written in Hyper Text Markup Language (HTML). HTML has three\nspecial characters:\n<, &, >.\nAn HTML command is enclosed in <...>, for example \n<p>, which is a paragraph separator, or <b>\n(\"begin bold\") and </b> (\"end bold\"). So the following\nHTML text:\n\n
\n
\n
\nThis sentence contains <b>bold</b> text.\n
\n
\n
\n\nproduces:\n\n

\n

\n
\nThis sentence contains bold text.\n
\n
\n

\n\nA Web page starts with a series of HTML commands, and ends with\na few more. The contents go in between:\n\n

\n

\n
\n
\n<!DOCTYPE HTML>\n<html lang=\"en\">\n<head>\n<META charset=\"UTF-8\">\n<META name=\"viewport\"\n content=\"width=device-width, initial-scale=1.0\">\n<title>Sample Web Page</title>\n</head>\n<body>\n
\n

\n(Contents go here)\n

\n

\n</body>\n</html>\n
\n
\n
\n

\n\nThe first line (DOCTYPE) specifies which markup\nlanguage the page uses\n(HTML = Hypertext Markup\nLanguage); just copy this line. The next\nline, <html lang=\"en\">, starts the page\nand specifies\nthe (human) language it is written in (language codes are specified\nhere),\nand is matched by the line \n</html>, which closes the\npage. <head> starts the heading, which\ncontains a title to be displayed on the browser's title bar and a\ndeclaration of the character set (nowadays it should always be\nUTF-8) and the \"viewport\"\nline which is a compulsory adaptation for cell phones, \"smart\" watches, etc.\n</head> closes the heading. The head can also\ncontain other items such as style parameters that you can learn about later;\nfor example by asking Google (\"HTML how do I change the font size?\").\n\n

\n\nThe <body> tag starts the body of the\ndocument, is closed by </body> tag.\n\n

\n\nAs you can see, most HTML commands come in begin-end pairs:\n<b>...</b>,\n<head>...</head>, etc. The closing part of the command\nhas a slash (/) between the < and the first letter of\nthe command.\n\n

\n\nBlank lines and line breaks are ignored. The browser automatically \"flows\"\nyour text into lines and paragraphs that fit in its window. Paragraphs must\nbe separated by <p>. Line breaks can be\nforced by <br>.\n\n

\n

\n
Example for Windows:\n\n
Use the mouse to copy the HTML above into NotePad. Then save the file\n(File -> Save As...)\nin your Web directory as index.html. Suppose your Windows username\nis Olga. Then (depending on which version of Windows you have) this might\nbe:\n\n
\n
\n
\nC:\\Users\\Olga\\Desktop\\Web\\index.html\n
\n
\n
\n\nNow to see your new web page, just double-click on the Web folder and\nthen double-click on index.html.\n
\n

\n\nNow you're ready to start adding \"content\" to your web page. Go back to\nNotePad and replace the title and \"(Contents go here)\" with whatever you\nwant. Any time you want to see the result, use\nFile -> Save in NotePad and then\nclick the Reload button on your browser.\n\n

\n\nThe next sections tell how to achieve different kinds of effects.\n\n

4. Converting Plain Text to HTML

\n\nIf you have a plain text file that you want to convert to HTML, load the\nfile into a plain-text editor and then follow these steps.\n\n

\n

    \n
  1. Change all occurrences of \"&\" to \"&amp;\".\n
  2. Change all occurrences of \"<\" to \"&lt;\".\n
  3. Change all occurrences of \">\" to \"&gt;\".\n
  4. Change any accented letters to HTML entity names (previous section)*.\n
  5. Put \"<p>\" between each paragraph.\n
  6. Insert the standard prolog at the top, substituting an appropriate title.\n
  7. Add </body> and\n </html> at the end.\n
  8. Save the result as xxx.html, where\nxxx is the part of the original file's name\nbefore the dot, or whatever-else-you-want-to-call-it.html.\n
\n

\n\nIf you are a Kermit user, you\ncan find a script to convert plain text to HTML\nHERE.\n\n

\n\nIf the text contains lists, tables, or other structures, read on.\n\n

\n\nIf you have a Microsoft Word document you want to convert to HTML, and your\ncopy of Word does not allow the file to be \"Saved As\" HTML, then save it as\nplain text and follow the same instructions. In this case you lose the\n\"richness\" (bold, italics, font changes, etc) when you save the file, and\nwill have to put the effects back by hand (next section).\n\n

\n\n\n
*\n Not necessary if your text is already encoded as UTF-8. If it's not\nUTF-8, you can identify the encoding in the <META charset=\"...\">\ndirective, but this topic is a bit advanced for this simple tutorial.\n
\n\n

7. Links

\n\nLinks can be internal within a Web page (like to\nthe Table of Contents at the top), or they\ncan be to external web pages or pictures on the same website, or they\ncan be to websites, pages, or pictures anywhere else in the world.\n\n

\n\nHere is a link to the Kermit\nProject home page. And here is what the HTML looks like:\n\n

\n
\n
\n<a href=\"http://www.kermitproject.org/\">\nKermit Project home page\n</a>\n
\n
\n
\n

\n\nThe part inside the quotes is called the URL (Uniform Resource Locator).\nHere is a link to Section 6 of the page you are\nreading, and the HTML:\n\n

\n
\n<a href=\"#lists\">Section 6</a>\n
\n
\n\nThe \"#\" indicates an internal section ID, in this case:\n\n
\n
\n<h3 id=\"lists\">6. Lists</h3>\n
\n
\n\n

\n\nHere is a link to\nSection 4.0 of\nanother document, at another website;\nthe C-Kermit for Unix\nInstallation Instructions. And the HTML:\n\n

\n
\n<a href=\"http://kermitproject.org/ckuins.html#x4.0\">\nSection 4.0\n</a>\n
\n
\n\n

\n\nHere is a link to a picture:\nCLICK HERE to see it.\n\n

\n\nIf you want to link to a particular section of somebody else's Web\npage, visit the page, \"view source\", search for the text at that spot\nand see if there is an \"id=\" clause; if so, use the ID as shown just above;\nif not you're out of luck.\n\n

\n\nIf you want to link to a particular page of a PDF document, just put\n\"#page=123\" (replace by the desired number) at the end of the URL.\n\n

9. Viewing Your Web Page

\n\nAnyway, back to basics. If you make a simple index.html\nin your Web directory like:\n\n
\n
\n<!DOCTYPE HTML>\n<html lang=\"en\">\n<head>\n<title>My first web page</title>\n<META charset=\"utf-8\">\n<META name=\"viewport\"\n content=\"width=device-width, initial-scale=1.0\">\n</head>\n<body>\n<h2>This is a heading</h2>\nAnd this is some text.\n</body>\n</html>\n
\n
\n\nThen if you double-click on index.html, it will open in your\nWeb browser.\n\n

\n\nNow you can work on your page's <body>: add more text,\nadd some images, add some links, add subheadings, some lists, some tables,\nwhatever you want. Each time you make a change, reload the page in your\nbrowser (usually done by clicking on the ⟳ symbol, or typing\nCtrl-R).\n\n

\n\nBy the way, a web page can have any name at all, it doesn't have to be\nindex.html. Index.html is a special name that is used for the \"home page\"\nof a website. To open a web page that has some other name, right-click on\nthe filename and then choose \"Open with...\"; then click on your Web\nbrowser's name.\n\n

10. Installing Your Web Page on the Internet

\n\nHow to put your web page on the Internet depends on your Internet Service\nProvider (ISP). At Columbia University, each user has a \"shell account\" on\nthe central server, which runs a Unix-based operating system, and which you\ncan access with a terminal emulator such as Kermit. Here's an example that\napplies to Columbia University's web server, showing how to upload your\nfiles from Windows:\n\n

\n

\n\n\nThere are easier ways to do this than what I describe below, but they\nrequire add-on software. The following method should work for everybody\nwho has Windows and an Internet connection.\n\n\n
\n

\n\nIf you create a public_html subdirectory of your\nlogin directory, give it \"world\" read and search permission, and then create\nan index.html file in that directory and give it world\nread permission, you'll have a home page. In this example\n\"$\" is the shell prompt (yours might be different),\nand what you type is underlined. CAUTION: the directory name\nis public_html but the underscore might be obscured\nthe underline in the examples below. Whenenever typing\n\"public_html\" always include the underscore.\nCAUTION#2: Some Web hosting sites might use different a different name for\nthe user's Web directory.\n\n

\n

\n
\n$ cd                      (Change to your login directory)\n$ mkdir public_html       (Create public_html subdirectory)\n$ chmod 755 public_html   (Give it world read/search permission)\n$ cd public_html          (Enter the public_html subdirectory)\n
\n
\n

\n\nYou only have to do this part once. Remember, it's public_html with\nan underscore, which tends not to show up when a command is underlined.\n\n

\n\nLet's assume you have created a website in the Web folder on your PC.\nHere's an example of how to upload your Web files to your public_html\ndirectory on Columbia University's\nCunix server using FTP (File Transfer Protocol).\nFirst start the FTP program:\n\n

\n

\nStart -> Run\n
\n

\n\nand type \"ftp\" in the box. An FTP window opens and an \"ftp>\"\nprompt appears. Type the underlined commands at the \"ftp>\"\nprompt (substituting your own user ID, etc):\n\n

\n
\n
\nftp> lcd Desktop\nLocal directory now C:\\Users\\olga\\Desktop.\nftp> lcd Web\nLocal directory now C:\\Users\\olga\\Desktop\\Web.\nftp> open cunix\nConnected to cunix.cc.columbia.edu.\n220 Cunix FTP server (Version 5.60) ready.\nUser (cunix.cc.columbia.edu:(none)): olga\n331 Password required for olga.\nPassword: (type your password here)\n230 User olga logged in.\nftp> cd public_html (\"public_html\" with underscore)\nftp> binary\nftp> put index.html\n200 PORT command successful.\n150 Opening BINARY mode data connection for index.html.\n226 Transfer complete.\nftp: 285 bytes sent in 0.00Seconds 285000.00Kbytes/sec.\nftp> site chmod 644 index.html\n200 CHMOD command successful\nftp> bye\n
\n
\n
\n\nThis sends the index.html file to your public_html\ndirectory on the server. You can send any other file by substituting its\nname for \"index.html. If you want to send all the files in\nyour Web folder, replace \"put index.html with \"put *\"\n(asterisk, meaning \"all files\" in this directory). Always use binary mode\nunless you know what you're doing.\n\n

\n\nIf the \"site chmod\" command failed (this service is not supported by some\nFTP servers), you have one more step. Before others can see your web files,\nyou have to give them \"world read\" permission. Again, log in to the server\nusing a terminal emulator (Telnet, SSH, Kermit, whatever), and:\n\n

\n

\n
\n$ cd ~/public_html        (Enter the public_html subdirectory)\n$ chmod 644 *             (Make all files publically readable)\n
\n
\n

\n\nNow you have a home page. If you were at Columbia and your login ID was\n\"olga\", the address (URL) of your home page would be:\n\n

\n

\nhttp://www.columbia.edu/~olga/\n
\n

\n\nIf you want to add pictures to your Web page, you can upload those too (also\nwith Kermit or FTP), and you also have to \"chmod 644\" all the files to make\nthem readable by everybody. Every time you add new files to your\npublic_html directory, you have to \"chmod 644\" them so they\nare accessible, either in the FTP session itself (as shown previously), or\nby logging in to the host and:\n\n

\n

\n
\n$ cd ~/public_html (\"public_html\" with underscore)\n$ chmod 644 *\n
\n
\n\n

\n\n\nPictures should be in JPG or PNG or GIF format. To include a picture\n(\"image\") in your page, include a sequence like this at the desired spot:\n\n

\n

\n
\n<img src=\"filename\" alt=\"brief description\">\n
\n
\n

\n\nReplace filename by the name of the file\n(e.g. skyline.jpg). Almost every HTML tag can be\ncustomized by \"attributes\" in the begin tag. For example if you want the\nimage to scale itself to the viewer's window (on a computer, cell phone,\nor other device), and furthermore you want the text of the page to flow\naround it, you can do:\n\n

<img alt=\"brief description\"\n style=\"width:50%;\n max-width:480px;\n float:left;\n margin:10px;\"\n src=\"filename\">\n
\n\n

\n\nYou can look up the attributes in Google, just search for html width,\n html float, etc.\n\n

\n\nNow you have your own home page on the Web, and your own URL (Uniform\nResource Locator, or Web address). In this example, the URL is:\n\n

\n

\nhttp://www.columbia.edu/~olga/\n
\n\n

\n\nOf course, if you prefer, you can also do all the Web-page editing directly\non the server, using an a server-based text editor like EMACS or Vi\nwhile logged in to the Unix shell. In that case you don't need to upload\nanything (except maybe photos), but then you also need to be more familiar\nwith the server's Unix environment and commands and utilities.\n\n

11. Where to go from here

\n\nMost Web pages are created by hideous bloated \"Web Authoring\" tools, which\nare usually designed to hook you (and readers of your web pages) into some\ncorporate profit-making scheme. If all you want is text with some pictures\nand links, some section headings, and maybe some tables, as opposed to\nspinning blinking popup holograms with streaming video, sound effects, etc,\nit's best to keep it simple and do it yourself. This is how the Web started\noff in the HTML 1.0 days of the early 1990s. The ingenious thing was that\nit was self propogating. If you saw a web page with a certain effect and\nwanted to know how it was done, you could simply \"view source\" to get the\n\"source code\" and then adapt it to your own page. You can still do that\nwith pages that look like this one, but since most Web pages are no longer\nmade by hand, you'll often see tons of incomprehensible gibberish (the more\nspecial effects, the more gibberish), for example\nat CNN.\n\n

\n\nAnyway, if you have mastered the simple techniques shown in this page, you\nknow the basics. Which is more than can be said of many \"web designers\" who\nonly know how to use prepackaged software to create web pages by picking\nthings from menus and moving things around with a mouse. To go further,\nyou can almost always find out how to do what you want by searching\nGoogle (\"html how do I ...?\"), or looking at the HTML code of different\nwebsites (browser \"view source\" command) but, again, only for pages\nthat look like this one.\n\n

\n\nOf course HTML is a standard, and here are the official references:\n\n

    \n
  • The\nHTML\n5.2 specification (2017).\n\n
  • Cascading\nStyle Sheets (yet another \"standard\" layered on top of HTML, which, like\nHTML goes through many versions, each one making the previous one obsolete).\n\n
\n\nYou can check the validity of your web page at\nthe W3C Markup Validation Service.\nNote that this page itself does not pass the Validator because it uses a\nnumber of \"obsolete\" elements. That's because (a) they are much easier to\nexplain, and (b) they still work. For the first 20 years or so HTML was in\nconstant flux, but with the release of HTML5 in 2014, it seems to be pretty\nstable.\n\n

\n\nIf you have made mistakes it will let you know, and if you have used \"old\"\nor \"deprecated\" HTML features it will let you know that too, and usually also\nsuggest a modern replacement.\n\n\n

12. Postscript: Cell Phones

\n\n
\nThe original Web was composed of pages designed to fit on desktop computer\nscreens, which, over time, became wider and wider. But then suddenly they\nalso had to fit on miniscule cell-phone and even \"smart watch\" screens. The\nmain pitfall is that an image might be too wide for the screen, so the image\nwidth should be specified as a percentage of the viewport width, e.g.:\n
\n\n
\n
\n
\n<img alt=\"Brief description\" \n title=\"Slightly longer description\"\n src=\"picture-of-something.jpg\"\n style=\"width:100%;\">\n
\n
\n
\n\nText, on\nthe other hand, usually just flows to accomodate the viewport. But if your\npage includes text that must not be \"wrapped\" (for example, program source\ncode, poetry, computer dialog transcripts), you have to enclose such\nsections within:\n\n

\n

\n<div style=\"overflow-x:auto; white-space:pre\">\n
\nmaterial that must not wrap\n
\n</div>\n
\n\n

\n\nas has been done in several places above, in which case a horizontal\nscroll bar will appear automatically if the non-wrapping text is wider than\nthe viewport. If you are viewing this page on a wide screen, you can\nsee this effect if you squeeze your browser horizontally to its minimum\nwidth and then scroll through this page.\n\n
